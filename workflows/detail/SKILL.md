# /detail — 需求详化

## 它做什么

原版交互全部保留（逐条追问，1-2 个问题，答了就过）。
区别：
- 原版：开放式追问 → 更新 spec/PRD.md
- 新版：**选项式追问**（按 GEARS 模式引导）→ 产出 `spec/detailed-requirements.md`
- 加料：追问方向按 GEARS 五种模式预设

---

## 流程

### 第一步：读输入（改）

读 `spec/confirmed-requirements.md`，提取需求清单 R1-Rx。

### 第二步：选项式逐条详化（改）

按 R1 → R2 → ... → Rx 顺序，每条最多问 **2 个问题**。

**不给开放式问题，给选项。** 每个选项包含"其他"兜底。

#### 追问方向模板（按需求类型匹配）

判断需求属于哪种类型，用对应的模板：

**类型 A — 用户操作类**（注册/登录/提交/创建）
```
1. 触发方式？
  A. 用户主动操作时触发（事件式）
  B. 系统自动执行（恒定式）
  C. 满足某个条件才触发（条件式）

2. 操作结果的状态流转？
  A. 提交中 → 成功 / 失败
  B. 提交中 → 处理中 → 成功
  C. 不需要状态，同步返回结果
```

**类型 B — 后台处理类**（审查/分析/计算/同步）
```
1. 处理方式？
  A. 同步处理，等结果返回
  B. 异步处理，完成后通知
  C. 后台静默执行，不通知

2. 出错怎么处理？
  A. 重试 N 次，都失败就报错
  B. 失败一次就跳过，记日志
  C. 失败后通知人工处理
```

**类型 C — 数据存储类**（保存/读取/查询/记录）
```
1. 数据存哪？
  A. 本地文件/数据库
  B. 云端服务
  C. 不存，临时内存

2. 数据谁可见？
  A. 只有自己
  B. 同项目成员
  C. 公开
```

**类型 D — 通知/展示类**（显示/通知/提醒）
```
1. 展示方式？
  A. 实时推送到界面
  B. 用户手动刷新查看
  C. 发通知（邮件/消息）

2. 什么时候更新？
  A. 有变化就更新
  B. 定时刷新
  C. 用户主动刷新
```

**类型 E — 通用追问**（无法匹配上面类型的兜底）
```
1. 这个功能的核心流程？
  A. [选项1]
  B. [选项2]
  C. 其他

2. 边界情况？
  A. [常见边界1]
  B. [常见边界2]
  C. 其他
```

**规则：**
- 每条需求只挑 **1-2 个最相关**的方向问，不四个全问
- 一次只问一个问题+选项，等回答再问第二个
- 用户选"其他"自己说了方案 → 记下来，过
- 用户说"不确定" → 记"待定"，过
- 不需要追问时（需求已经很明确）→ 直接过

### 第三步：产出 detailed-requirements.md（改）

写 `spec/detailed-requirements.md`。

格式：

```
R1: 用户能用邮箱和密码注册新账号
  - 触发: 事件式（用户提交注册表单时触发）
  - 条件: 邮箱未注册 / 密码 ≥8 位含字母数字
  - 状态: 空闲 → 提交中 → 成功 / 失败
  - 错误:
    - 邮箱已注册 → 提示"该邮箱已注册"
    - 密码太短 → 提示"密码需 ≥8 位"
    - 网络错误 → 提示"请稍后重试"
  - 待定: 密码找回功能（第一期不确定做不做）

R2: 用户能用邮箱和密码登录
  - 触发: 事件式（用户提交登录表单时触发）
  - 条件: 邮箱已注册 + 密码正确
  - 状态: 空闲 → 登录中 → 成功 / 失败
  - 错误:
    - 密码错误 → 提示"密码错误，还剩 N 次机会"
    - 账号不存在 → 提示"该邮箱未注册"
  - 状态: <空闲> → <登录中> → <已登录> / <失败>
```

**没覆盖到的开放式问题/待定项，统一标在末尾：**

```
### 未定项
- R1 密码找回: 第一期不做，二期再看
- R3 支持哪些语言审查: 第一期只做 TypeScript
```

### 第四步：输出

```
✅ 需求详化完成 → spec/detailed-requirements.md

R1（注册）: 事件式，条件邮箱未注册+密码≥8位
R2（登录）: 事件式，条件邮箱已注册+密码正确
R3（审查）: 事件式，异步处理+结果贴回

下一步: /draft-spec 写技术方案
```

- 一条需求最多问 2 个问题
- 不问已经确认过的东西
- 你答了就过，不循环确认
- 你说"不确定"，记"待定"，过
