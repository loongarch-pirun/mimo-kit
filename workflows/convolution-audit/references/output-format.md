# 产出格式模板

## 1. 每维产出完整格式

```
==================================================================
维度 {ID}：{名称}  |  轮次：{1/2/3}  |  状态：{收敛通过 / 登记盲区}
==================================================================

## CLI 硬证据（{工具名} v{版本}）

{原始输出摘要 ≤10 行，或标注"本维无 CLI 前置"}

来源：【CLI 已确认：{工具名} v{版本}】
```

## LLM Agent 分析

{agent 名称} 分析结果：

### ■ 锚定发现（CLI 确认 / 代码直接可见）

| # | 发现 | 坐标 | 来源 |
|---|------|------|------|
| 1 | {描述 ≤1 行} | {文件:行号:函数} | CLI 确认 / 代码可见 |
| 2 | ... | ... | ... |

### ◆ 模式发现（LLM 识别跨文件）

| # | 发现 | 涉及文件 | 模式描述 |
|---|------|---------|---------|
| 1 | {描述 ≤1 行} | {N 个文件} | {模式名/类型} |
| 2 | ... | ... | ... |

## 收敛检查清单

轮次：{1/2/3}

| # | 检查项 | 状态 | 证据/坐标 |
|---|--------|------|----------|
| 1 | （具体项）| ✓ | {文件:行号 / CLI 输出} |
| 2 | （具体项）| ✗ | 未通过原因 |
| 3 | （具体项）| ○ | 第 3 轮仍未确认 |

通过：{N}/{总数}  盲区：{M}/{总数}

本轮结果：□ 收敛通过（继续下一维） □ 登记盲区（3 轮耗尽）

## 摘要卡

收敛状态：✓ 通过（{N}/{M} 项） / ○ {N} 项盲区
硬证据（CLI）：{工具名} → {关键结果 1-2 条}
锚定发现（■）：{N} 条（关键：{3 条，含坐标}）
模式发现（◆）：{N} 条（关键：{2 条}）
盲区（○）：{N} 条（{列表} 或 "无"）
→ 传给下一维关注：{1-3 条}

==================================================================
```

## 2. 摘要卡模板（≤15 行）

```
## 摘要卡：维度 {ID} {名称}
收敛状态：✓（{N}/{M}）/ ○ {N} 盲区
CLI：{工具} → {1-2 条关键结果}
■：{N}条，{最优 3 条含坐标}
◆：{N}条，{最优 2 条}
○：{N}条，{列表/无}
→ 下维关注：{1-3 条}
```

**行数限制**：上述模板填充后必须 ≤15 行。超限时裁减发现数量（保留权重最高的），不缩小坐标格式。

## 4. 一页纸摘要模板（final-report.md 末尾）

```markdown
# 一页纸摘要

## 项目
{一句话总结}

## 6+2 维度覆盖矩阵

| 0 | A | B | C | E | F | K | L |
|---|---|---|---|---|---|---|---|
| ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | — | — |

✓ 收敛通过  ○ 含盲区  ✗ 未通过  — 未触发

## 关键发现

### ■ 锚定（CLI 确认）
{N} 条硬证据发现，Top 3：
1. {发现 + 坐标}
2. {发现 + 坐标}
3. {发现 + 坐标}

### ◆ 模式（跨文件）
{N} 条模式发现，Top 2：
1. {模式描述}
2. {模式描述}

### ○ 盲区
{N} 项不能确认，详见 blind-spots.md

## 关键风险（最多 2 条）

1. **{风险}**：{具体说明 + 坐标}
2. **{风险}**：{具体说明 + 坐标}

## 下一步

{1 行建议，如：建议运行 security-reviewer 跟进 ■ 锚定发现 / 建议运行 cognitive-map 扫描补充语义视角}
```

|------|---------|---------|------|
| 0 | ✓ 收敛 | ✓ 收敛 | 无新发现 |
| B | ✓ 收敛 | ○ 1 盲区 | 新增逻辑路径未确认 |
| ... | ... | ... | ... |

新发现：
- {发现}

新盲区：
- {盲区}

未变化维度沿用上次摘要卡（{上次审计日期}）。
```
